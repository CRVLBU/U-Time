# ⚙️ UtimeAPI

**UtimeAPI** — это функциональный бэкенд-модуль, обеспечивающий взаимодействие веб-приложения с базой данных. Он выступает в роли промежуточного слоя между пользовательскими контроллерами и СУБД, обрабатывая запросы (сформированные контроллерами [UtimeWEB](../Utime_WEB/README.md)), и возвращая структурированные данные.

## 🧱 Структура директорий

| Директория | Назначение |
|------------|------------|
| `Controllers/` | Контроллеры, которые принимают запросы от клиента и возвращают ответы, используя данные из репозиториев. |
| `Data/` | Содержит классы контекста базы данных (`DbContext`) и другие файлы конфигурации подключения. |
| `Migrations/` | Автоматически сгенерированные файлы миграций для синхронизации схемы базы данных с моделями. |
| `Models/` | модели, представляющие таблицы базы данных. |
| `Repository/` | Репозитории для работы с базой данных  |

## 🛠 Технологический стек

### 🗄️ СУБД: Microsoft SQL Server

Выбор пал на **Microsoft SQL Server** благодаря ряду весомых преимуществ:

- 🔗 **Глубокая интеграция с .NET и Entity Framework Core** — максимальная совместимость и удобство разработки.
- ⚡ **Высокая производительность и расширяемость** — эффективная работа с большими объёмами данных и сложными запросами.
- 🔒 **Надёжность, безопасность и корпоративные функции** — шифрование, резервное копирование, поддержка транзакций и механизмы высокой доступности.

### 🛠 Функциональные модули системы (API)
Программный интрфейс разделён на логические блоки, каждый из которых отвечает за конкретный аспект веб-сайта:

### ✔ Activity - задачи
Создание, обновление, получение и удаление задач. Изменение статуса задачи, получение данных для статистики о выполнении пользователем задач.

### 📒 Category - категории
Создание, обновление, получение и удаление категорий. Подсчёт количества задач, принадлежащих каждой категории.

### 🔐 User - пользователь
Регистрация, вход в аккаунт пользователя. Генерация JWT-токена для аутентификации с помощью класса JwtSecurityTokenHandler.
