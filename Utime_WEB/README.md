# 🌐 UtimeWEB

**UtimeWEB** — это раздел, отвечающий за взаимодействие с пользователем, включающий  в себя контроллеры, обрабатываюobt пользовательский ввод и создающие запросы к контроллерам [UtimeAPI](../UtimeAPI/README.md). Также данный блок отвечает за возвращение представлений, в качестве Razor-страниц пользователю.

## 🧱 Структура директорий

| Директория | Назначение |
|------------|------------|
| `Controllers/` | Контроллеры, которые обрабатывают ввод от клиента и возвращают ответы в виде представлений. |
| `Services/` | Содержит сервисы для формирования запросов к контроллерам [UtimeAPI](../UtimeAPI/README.md). |
| `Views/` | Содержит файлы представлений. |
| `Models/` | модели, представляющие таблицы базы данных. |
| `wwwroot/` | Содержит статические файлы (СSS, JavaScript) для страниц веб-сайта.  |

## 🛠 Технологический стек

### 🎨 Инструменты для Frontend
- **Движок Razor**
      Позволяет встраивать C# код непосредственно в HTML-разметку для создания динамических веб-страниц.

       Создание полноценных, функциональных и привлекательных пользовательских интерфейсов.

### 🛠 Функциональные модули системы (API)
Программный интрфейс разделён на логические блоки, каждый из которых отвечает за конкретный аспект веб-сайта:

### ✔ Activity - задачи
Создание, обновление, получение и удаление задач. Изменение статуса задачи (выполнен/не выполнен), отображение длительности задач, сортировка задач по приоритету.

### 📒 Category - категории
Создание, обновление, получение и удаление категорий. Подсчёт количества задач, принадлежащих каждой категории.

### 🔐 User - пользователь
Регистрация, вход в аккаунт пользователя. Создание аутентификационных кук на основе JWT-токена, полученного после обращения к контроллеру [UtimeAPI](../UtimeAPI/README.md).

### 📊 Account - статистика выполнения задач
Отображение статистики количества выполненных, невыполненных и выполненных с опозданием задач за последние 7 дней, по отображение статистики выполнения по приоритетам.
